Этап 3 — Расширение форума

Комментарии к отзывам
- Таблица `comments` (id, review_id, user_id, body, parent_id, created_at)
- `parent_id` — для вложенных ответов (ответ на ответ)
- Отображение дерева комментариев под отзывом
- Форма быстрого ответа прямо в окне отзыва
- Счётчик комментариев на карточке в списке
- Кнопка «Ответить» под каждым комментарием

Лайки / «Полезно»
- Таблица `review_likes` (id, review_id, user_id) — уникальная пара
- Кнопка «👍 Полезно» на карточке и в детальном окне
- Счётчик лайков рядом с кнопкой
- Один пользователь — один лайк на отзыв (повторный клик убирает)
- Сортировка по популярности (кол-во лайков)

Фильтр по стране и городу
- Два отдельных поля: выпадающий список стран + поле города
- Страны подгружаются из реальных данных отзывов (`DISTINCT country`)
- При выборе страны список городов обновляется автоматически
- Сброс фильтров одной кнопкой «✕ Очистить»

 Сортировка
- Радио-кнопки или выпадающий список: По дате ↓ / По дате ↑ / По рейтингу / По популярности (лайки)
- Сортировка применяется вместе с фильтрами без перезапуска

Галерея фото в отзыве
- Миниатюры фото (thumbnails 80×80) под текстом отзыва
- Клик на миниатюру → полноэкранный просмотр
- Навигация вперёд/назад между фото в одном отзыве
- Масштабирование колёсиком мыши
- Генерация миниатюр через `Pillow` при сохранении фото

Теги
- Таблица `tags` (id, name) + `review_tags` (review_id, tag_id)
- Поле ввода тегов через запятую в форме отзыва
- Автодополнение при вводе из существующих тегов
- Отображение тегов на карточке в виде цветных «чипсов»
- Клик на тег → фильтрация всех отзывов с этим тегом

Этап 4 — Расширение базы мест

 Статичная карта (OpenStreetMap)
- Поля `latitude` и `longitude` в таблице `places`
- При добавлении места — кнопка «Получить координаты» по адресу (Nominatim API)
- Загрузка тайла карты через `requests` + OpenStreetMap Static Maps
- Отображение карты в карточке места как изображение (400×250px)
- Маркер точки на карте

Галерея фото места
- Собирает все фото из привязанных отзывов (`review_photos JOIN reviews`)
- Горизонтальная полоса миниатюр в карточке места
- Клик → полноэкранный просмотр с навигацией
- Подпись под фото: автор отзыва + дата

Подтверждённые места
- Поле `verified BOOLEAN` в таблице `places`
- Значок «✓ Проверено» на карточке
- Отдельный фильтр «Только проверенные»
- Только сам создатель или любой пользователь может отметить (на выбор)

Экспорт карточки в PDF
- Библиотека `reportlab` или `fpdf2`
- Экспортирует: название, адрес, категория, рейтинг, описание, список отзывов
- Кнопка «⬇️ Скачать PDF» в карточке места
- Диалог выбора папки сохранения
- Логотип/шапка «ТревелХаб» в PDF

Этап 5 — Профили и сообщество

 Профиль пользователя
- Поля в `users`: `avatar_path`, `bio`, `home_country`, `home_city`
- Отдельный экран профиля (новая вкладка или модальное окно)
- Загрузка аватара из файла (сохраняется в `assets/avatars/`)
- Отображение аватара в статус-строке главного окна
- Список всех отзывов пользователя на его странице

Статистика пользователя
- Кол-во отзывов всего
- Средняя оценка в отзывах
- Кол-во уникальных стран в отзывах
- Кол-во лайков на отзывах пользователя
- Топ-3 категории (по кол-ву отзывов)
- Визуализация через `matplotlib` — столбчатая диаграмма активности по месяцам

Подписки и рекомендации
- Таблица `subscriptions` (follower_id, followed_id)
- Кнопка «Подписаться» на странице пользователя
- Лента «Подписки» — отзывы только от людей, на которых подписан
- Рекомендации: «Люди, которые писали о тех же странах»

Этап 6 — Технические улучшения

Пагинация
- Загрузка по 20 записей за раз
- Кнопки «← Назад» / «Вперёд →» внизу списка
- Отображение: «Страница 2 из 7»
- Или «бесконечная прокрутка» — подгрузка при достижении конца списка

Кеширование запросов
- Простой in-memory кеш (`dict` с TTL)
- Кешируются тяжёлые запросы: список мест с рейтингами, категории
- Инвалидация кеша при добавлении/изменении записи
- Время жизни кеша: 30–60 секунд

Экспорт данных
- Экспорт всех отзывов в `.csv` (через `csv` stdlib)
- Экспорт всех мест в `.json`
- Выбор папки сохранения через `filedialog`
- Кнопки в меню «Файл» или в настройках
- Импорт из JSON — добавление мест пачкой

Резервное копирование `travel.db`
- Кнопка «Создать резервную копию» → сохраняет `travel_backup_YYYY-MM-DD.db`
- Автобэкап при запуске раз в 7 дней (проверка даты последнего бэкапа)
- Хранение последних 5 бэкапов (старые удаляются)
- Кнопка «Восстановить из бэкапа»

Настройки приложения
- Отдельная вкладка или окно настроек
- Тема оформления: Светлая / Тёмная (переключение цветовых схем)
- Путь к папке фото (по умолчанию `assets/photos/`)
- Язык интерфейса (RU / EN — через словарь строк)
- Кол-во записей на странице (для пагинации)
- Сохранение настроек в `config.json`

Структура базы данных

users              — id, name, created_at

forum_categories   — id, icon, name

place_categories   — id, icon, name

places             — id, name, address, city, country, description,

                     working_hours, price_range, category_id

reviews            — id, user_id, place_id, title, body, rating,

                     category_id, country, city, created_at

review_photos      — id, review_id, file_path

Связи:

•  reviews.user_id → users.id

•  reviews.place_id → places.id 

•  reviews.category_id → forum_categories.id

•  places.category_id → place_categories.id

•  review_photos.review_id → reviews.id
